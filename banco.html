Banco de Dados

Essa sessão documenta as atividades e projetos realizados na disciplina de Banco de Dados, abordando desde conceitos básicos até a implementação prática em diferentes sistemas de banco de dados.

Durante o curso, exploramos conceitos fundamentais de dados, técnicas de modelagem, tipos de armazenamento e diferentes paradigmas de bancos de dados, tanto relacionais quanto NoSQL.

Introdução

No início do curso, discutimos a diferença entre dados e modelagem de dados. Entendemos que dados são fatos brutos e não processados, enquanto a modelagem de dados é uma técnica para organizar esses dados de maneira estruturada e lógica, facilitando o armazenamento, acesso e manipulação em sistemas de informação.

Dados vs Modelagem de Dados

Aprendemos que dados são unidades de informação que podem ser observadas e registradas. Porém, para transformar esses dados em informações úteis, é necessário organizá-los de maneira lógica. A modelagem de dados é a técnica que utilizamos para criar representações abstratas dos dados, que guiam a estruturação e o design de bancos de dados.

Estrutura de Dados

Estudamos a estrutura interna dos dados e como eles são organizados nos sistemas de banco de dados. Isso inclui a forma como os dados são armazenados em disco, a estrutura de índices para melhorar a performance de consultas e as técnicas de particionamento para gerenciar grandes volumes de dados de maneira eficiente.

Modelos de Modelagem de Dados

Modelo Conceitual

O modelo conceitual de dados é uma representação abstrata de como os dados são organizados. Utilizamos diagramas de entidade-relacionamento (ER) para mapear entidades, atributos e relacionamentos. Este modelo é independente do sistema de gerenciamento de banco de dados (SGBD) e serve como uma base para o desenvolvimento posterior de modelos lógicos e físicos.

Modelo Lógico

Transformamos o modelo conceitual em um modelo lógico, detalhando as tabelas, colunas e relações de um banco de dados relacional. Esta etapa envolve a definição de chaves primárias e estrangeiras, normalização dos dados e a preparação para a implementação física. O modelo lógico ainda é independente da tecnologia de armazenamento, mas está mais próximo da estrutura que será usada no SGBD.

Modelo Físico

No modelo físico, implementamos o modelo lógico em um SGBD específico. Este modelo define a estrutura exata das tabelas no banco de dados, incluindo tipos de dados, índices e partições. Também considera aspectos de desempenho, como otimização de consultas e armazenamento eficiente dos dados.

Tipos de Armazenamento

Exploramos diferentes tipos de armazenamento de dados, incluindo bancos de dados relacionais e não relacionais (NoSQL). Cada tipo tem suas vantagens e desvantagens, dependendo dos requisitos de aplicação, como integridade dos dados, performance, escalabilidade e flexibilidade.

Bancos de dados relacionais utilizam tabelas e SQL para gerenciar dados estruturados, enquanto bancos de dados NoSQL oferecem uma variedade de modelos de dados (documentos, grafos, colunas e chave-valor) para atender a diferentes necessidades de aplicações modernas.

Banco de Dados Relacionais

Estudamos bancos de dados relacionais, focando no uso do modelo entidade-relacionamento (ER) para modelagem de dados e na implementação de modelos físicos. Aprendemos sobre chaves primárias e estrangeiras, normalização e desnormalização, e técnicas para otimizar a performance de consultas SQL.

Os bancos de dados relacionais são amplamente utilizados devido à sua capacidade de garantir a integridade dos dados e realizar transações ACID (Atomicidade, Consistência, Isolamento e Durabilidade).

Modelo Entidade-Relacionamento (ER)

Utilizamos o modelo ER para criar diagramas que representam entidades e seus relacionamentos. Estes diagramas ajudam a visualizar e projetar a estrutura do banco de dados, identificando entidades, atributos, chaves primárias e relacionamentos (um para um, um para muitos, muitos para muitos).

O modelo ER é uma ferramenta crucial no design de bancos de dados relacionais, proporcionando uma visão clara de como os dados se inter-relacionam.

Modelo Físico (Baixo Nível)

Implementamos o modelo físico no banco de dados, detalhando a estrutura de tabelas, índices e otimizações de desempenho. Esta fase inclui a criação de tabelas, definição de tipos de dados apropriados para cada coluna, e implementação de restrições para garantir a integridade dos dados.

Também abordamos a importância de índices para melhorar a performance de consultas e técnicas de particionamento para gerenciar grandes volumes de dados.

Normalização

Aprendemos sobre a normalização de dados, uma técnica para eliminar redundâncias e garantir a integridade dos dados em um banco de dados relacional. A normalização envolve a divisão de grandes tabelas em tabelas menores e a definição de relacionamentos entre elas para minimizar a redundância e dependências anômalas.

Estudamos as diferentes formas normais (1FN, 2FN, 3FN, BCNF) e como aplicá-las para criar uma estrutura de banco de dados eficiente e consistente.

Arquivos de Aula

Os arquivos das aulas incluíram scripts SQL, exemplos de modelagem de dados e estudos de caso práticos. Esses recursos foram fundamentais para entender e aplicar os conceitos aprendidos, permitindo a prática de criação, manipulação e consulta de bancos de dados.

Banco de Dados NoSQL

Exploramos os conceitos e tipos de bancos de dados NoSQL, uma alternativa aos bancos de dados relacionais que oferece flexibilidade e escalabilidade para aplicações modernas. Aprendemos que os bancos de dados NoSQL são ideais para grandes volumes de dados não estruturados e para aplicações que requerem alta performance e disponibilidade.

Conceitos

Os bancos de dados NoSQL são projetados para lidar com grandes quantidades de dados e operações de leitura/escrita rápidas. Eles oferecem modelos de dados flexíveis e são capazes de escalar horizontalmente, o que é essencial para aplicações web e móveis.

Tipos de Bancos de Dados NoSQL

Estudamos os quatro principais tipos de bancos de dados NoSQL:

  • Document Stores: Armazenam dados como documentos JSON ou BSON (e.g., MongoDB).
  • Key-Value Stores: Armazenam dados como pares chave-valor (e.g., Redis).
  • Column Stores: Armazenam dados em colunas em vez de linhas (e.g., Apache Cassandra).
  • Graph Databases: Armazenam dados em grafos, enfatizando as relações entre os dados (e.g., Neo4j).

Atlas MongoDB

Utilizamos o Atlas MongoDB para explorar um banco de dados NoSQL na prática. Atlas MongoDB é uma plataforma de banco de dados como serviço (DBaaS) que facilita a implantação, operação e escalabilidade de MongoDB na nuvem.

Aprendemos como criar clusters, importar dados e executar consultas utilizando a interface web intuitiva do Atlas. Esta experiência prática nos permitiu compreender melhor como os bancos de dados NoSQL podem ser usados para resolver problemas de dados em aplicações reais.
